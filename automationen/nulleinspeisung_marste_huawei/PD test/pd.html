Interaktive Erklärung des Proportional-Derivative-Reglers für die Home Assistant Automation

Was ist ein PD-Regler?

Ein Regler hat eine einfache Aufgabe: Er misst einen Fehler (hier: Netzbezug laut Shelly in Watt) und steuert eine Stellgröße (hier: Marstek-Entladeleistung), um den Fehler auf null zu bringen. Der PD-Regler macht das mit zwei Anteilen, die zusammenaddiert werden:

P – Proportional

Wie weit bin ich jetzt daneben?

Schaut nur auf den aktuellen Fehler. Shelly zeigt +400 W Netzbezug? Dann fordert P genau Kp × 400 = 320 W mehr Entladeleistung. Einfach, reaktionsschnell — aber allein kann P überschwingen.

P = Kp × error
D – Derivative

Wie schnell ändert sich der Fehler?

Schaut auf die Änderungsrate des Fehlers. Fehler sinkt bereits schnell? D bremst P ab, bevor Marstek zu weit regelt. Fehler wächst explosiv? D gibt sofort Gas. Verhindert Pendeln ohne Integral-Aufwand.

D = Kd × (error − prev_error) / dt
P + D zusammen

Schnell und trotzdem stabil

Die Stellgröße ist die Summe beider Anteile, addiert auf den aktuellen Ausgabewert (Velocity-Form). Ein Rate-Limiter und ein Sprungmodus bei großen Fehlern (>400 W) sorgen für robustes Verhalten.

Δu = P + D → new_out = out + Δu

Tipp: Stelle Kd = 0 ein (reiner P-Regler) und drücke auf „Lastsprung" — beobachte wie die rote Linie nach dem Sprung um die Nulllinie pendelt. Erhöhe dann Kd auf 0.3–0.5 und wiederhole: das Pendeln verschwindet, weil D das Überschwingen vorausahnt und abbremst. Je größer Kp, desto schneller — aber auch desto mehr Neigung zum Schwingen ohne ausreichendes Kd.

Vollständige Formel (wie in der HA-Automation)

error        = shelly_eff + max(−emma, 0)     // EMMA-Priorität
derivative   = (error − prev_error) / dt       // D-Term
raw_adj      = Kp × error + Kd × derivative   // P + D
rate_adj     = clamp(raw_adj, −700, +400)     // asymm. Rate-Limit
// Sprungmodus bei großem Fehler:
new_output   = current + 0.8×error            // wenn |error| > 400 W
new_output   = current + rate_adj             // sonst (Velocity-Form)
new_output   = clamp(new_output, 0, 2500)     // Hardware-Grenze
